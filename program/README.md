# Мониторинг волатильности FX (валютного рынка)

Конвейер данных по мониторингу 10 валютных пар: ежедневная загрузка котировок
из Yahoo Finance, прогноз волатильности продвинутой моделью **GJR-GARCH(1,1)
со скошенным t-распределением + онлайн-калибровка Ridge на HAR-признаках**,
хранение в PostgreSQL и публикация результатов через REST API и
интерактивный дашборд.

Сервис разворачивается одной командой `docker compose up -d` и не требует
авторизации для просмотра дашборда.

## Что внутри

* **Целевые валютные пары (10):** EUR/USD, GBP/USD, USD/JPY, USD/CHF,
  AUD/USD, USD/CAD, NZD/USD, EUR/GBP, USD/RUB, EUR/RUB.
* **Дополнительные признаки (по 2 на пару):** список из ноутбука с
  `CorrelationFilter` — например, для EUR/USD это `DX-Y.NYB` и `HYG`,
  для USD/JPY — `^N225` и `^TNX`, и т.д.
* **Модель:** GJR-GARCH(1,1) со скошенным t-распределением (Hansen 1994),
  обучаемая на скользящем окне 252 дня, плюс онлайн Ridge-калибровка на
  HAR-признаках (лаги реализованной волатильности 1, 5, 21 день) на
  скользящем окне 500 точек. При несходимости MLE — fallback на EWMA
  (λ=0.94). Реализация: `shared/models.py`.
* **Индекс волатильности 1–100:** expanding min-max-нормализация
  реализованной волатильности; первые 100 точек исключаются, чтобы убрать
  «дёрганье» из-за малого окна (`WARMUP_BURN`).
* **Прогноз на завтра:** одношаговый прогноз продвинутой модели,
  отображается отдельным KPI и пунктирной линией на графике индекса.

## Запуск

### 1. Подготовка

```bash
git clone <this-repo> 
cd vkr
cp .env_example .env
# (опционально) поправьте .env — в первую очередь DATA_START_DATE
```

> Для Linux рекомендуется задать `AIRFLOW_UID=$(id -u)` в `.env`,
> чтобы директория `airflow/logs` создавалась с правильными правами.
>
> На macOS/Windows эту переменную можно оставить пустой.

### 2. Поднять весь стек

```bash
docker compose up -d --build
```

Первый запуск собирает образы (~3–5 минут).

### 3. Загрузить исторические данные и обучить модель

Откройте Airflow UI: **http://localhost:8080** - для локального запуска
(логин/пароль из `.env`, по умолчанию `admin / admin`).

Включите DAG `fx_pipeline` и запустите его вручную (Trigger DAG).
Шаги DAG-а:

1. `load_prices` — скачивает все тикеры (10 пар + 19 контекстных
   инструментов) из Yahoo Finance и пишет в `prices` через UPSERT.
2. `predict_volatility` — для каждой пары обучает GJR-GARCH +
   Ridge-калибровку на скользящем окне и пишет прогнозы в `predictions`.

Первый прогон делает полный backfill и может занять **20–60 минут** в
зависимости от `DATA_START_DATE`. Последующие ежедневные запуски
работают инкрементально (≈1–2 минуты).

DAG также запускается автоматически каждый день в **22:00 UTC**
(после закрытия NYSE).

### 4. Открыть дашборд

После завершения DAG-а зайдите на **http://localhost:8501**.
Авторизация не требуется.

## Дашборд

* **Сайдбар:** селектор валютной пары (10 пар), выбор периода (1м / 3м / 6м /
  1г / 3г).
* **Сверху:** KPI-карточки — последняя цена и её дневное изменение,
  20-дневная реализованная волатильность, текущий индекс волатильности
  (1–100) и прогноз индекса на следующий торговый день.
* **Главный график:** курс закрытия выбранной пары.
* **Гейдж + история индекса волатильности 1–100:** жёлтая зона 30–70,
  красная 70+; пунктирная линия — прогноз модели на следующий день.
* **Снизу:** два компактных графика дополнительных признаков (DXY/HYG и
  т.п. в зависимости от пары) с дневным изменением.

Тема: бело-синяя (`primaryColor=#1f4e79`, `backgroundColor=#FFFFFF`),
задаётся в `dashboard/.streamlit/config.toml`.

## REST API

Swagger: **http://localhost:8000/docs**

| Метод | Путь                          | Параметры              | Назначение                              |
|-------|-------------------------------|------------------------|-----------------------------------------|
| GET   | `/health`                     | —                      | проверка БД                             |
| GET   | `/pairs`                      | —                      | список пар и контекстных тикеров        |
| GET   | `/history/{pair}`             | `start`, `end` (даты)  | OHLC по паре                            |
| GET   | `/context/{pair}`             | `start`, `end`         | OHLC дополнительных признаков пары      |
| GET   | `/predictions/{pair}`         | `start`, `end`, `limit`| исторические прогнозы и индекс          |
| GET   | `/latest/{pair}`              | —                      | сводка для KPI-карточки                 |
| GET   | `/etl/stats`                  | `days`                 | статистика запусков pipeline-а          |

`pair` передаётся со слешем, например `/latest/EUR/USD`.

## Замечания по отказоустойчивости

* Все запросы yfinance и MLE-фиты GARCH обёрнуты в `try/except`
  с фолбэком на EWMA — pipeline не падает на отдельных тикерах/окнах.
* `prices` и `predictions` обновляются через `INSERT ... ON CONFLICT
  DO UPDATE` — повторный запуск идемпотентен.
* В таблице `etl_log` сохраняется статус каждого прогона
  (`/etl/stats` его читает).
* Airflow настроен на `retries=2`, `retry_delay=5min`.
* Healthcheck-и Postgres и Airflow webserver — встроены в `docker-compose.yml`.
